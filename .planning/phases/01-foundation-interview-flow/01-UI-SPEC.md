# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation + Interview Flow.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (to be initialized — greenfield project, no components.json yet) |
| Preset | Run `npx shadcn@latest init` with default "New York" style, neutral base color |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react (ships with shadcn/ui) |
| Font | Inter (system-ui fallback) — loaded via `next/font/google` |

shadcn/ui components to add at init:
`npx shadcn@latest add button card textarea badge accordion separator`

No third-party registries. Official shadcn only.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding, status dot margin |
| sm | 8px | Button icon gap, badge padding, inline element spacing |
| md | 16px | Card internal padding, form field gap, sidebar item padding |
| lg | 24px | Section padding, question text margin-bottom |
| xl | 32px | Panel vertical padding, layout column gap |
| 2xl | 48px | Setup card vertical padding, major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for domain chips in sidebar: minimum 44px height (accessibility requirement)
- Textarea minimum height: 120px (multi-sentence answers)

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Page title on setup screen only |
| Heading | 24px | 600 (semibold) | 1.3 | Current interview question text |
| Body | 16px | 400 (regular) | 1.6 | Answer textarea, probe suggestion text, form labels |
| Label | 14px | 400 (regular) | 1.4 | Domain names in sidebar, status chips, auto-save indicator, button text |

Font stack: `Inter, ui-sans-serif, system-ui, sans-serif`

Rules:
- Question text (Heading/24px) must never be truncated — allow full wrap
- Answer textarea uses Body/16px at weight 400 — comfortable for reading back long answers
- No font sizes below 14px anywhere in Phase 1 UI

---

## Color

Palette rationale: calm, clinical-adjacent, not sterile. Warm neutral base to soften the end-of-life context.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF9` (warm white / stone-50) | Page background, main content area background |
| Secondary (30%) | `#F5F4F2` (stone-100) | Left sidebar background, setup card background, probe panel background |
| Surface | `#FFFFFF` | Answer textarea background, active question card |
| Border | `#E7E5E4` (stone-200) | Card borders, dividers, textarea border |
| Text primary | `#1C1917` (stone-900) | Question text, answer text, headings |
| Text secondary | `#78716C` (stone-500) | Domain labels, placeholder text, auto-save timestamp |
| Accent (10%) | `#0EA5E9` (sky-500) | Primary CTA buttons ("Begin Interview", "Next Question"), active domain indicator, probe chip hover state |
| Accent muted | `#E0F2FE` (sky-100) | Probe chip background (resting state) |
| Success | `#16A34A` (green-600) | Domain "complete" status chip, "Saved" indicator text |
| Destructive | `#DC2626` (red-600) | "Skip Domain" confirmation text only |

Accent reserved for:
- "Begin Interview" button (setup screen CTA)
- "Next Question" button (primary interview action)
- Currently active domain indicator in sidebar (left border highlight)
- Probe chip hover/focus state

Accent is NOT used for: decorative elements, backgrounds, domain status chips, navigation links.

Dark mode: not in scope for Phase 1. All values are light-mode only.

## Interaction Design

### States

| Element | State | Visual |
|---------|-------|--------|
| Domain row (sidebar) | Default | stone-100 bg, stone-700 text |
| Domain row (sidebar) | Active | sky-500 left border (4px), stone-50 bg |
| Domain row (sidebar) | Hover | stone-200 bg, cursor pointer |
| Domain row (sidebar) | Focus | 2px sky-500 outline, offset 2px |
| Answer Textarea | Default | white bg, stone-200 border |
| Answer Textarea | Focus | sky-500 border (2px), no box-shadow |
| Answer Textarea | Filled | stone-900 text |
| Answer Textarea | Empty + Next clicked | red-500 border + "Please enter an answer before continuing" below |
| Next Question button | Default | sky-500 bg, white text |
| Next Question button | Hover | sky-600 bg |
| Next Question button | Disabled (no answer) | stone-300 bg, stone-400 text, cursor not-allowed |
| Next Question button | Loading (AI probe firing) | spinner icon left, "Saving..." text, disabled |
| Probe chip | Default | sky-100 bg, sky-700 text, rounded-full |
| Probe chip | Hover | sky-200 bg |
| Probe chip | Focus | 2px sky-500 outline |
| Probe chip | Clicked | stone-100 bg, stone-500 text (used state) |
| AutoSaveIndicator | Saving | opacity 0.5 |
| AutoSaveIndicator | Saved | opacity 1, green-600 text, "Saved · HH:MM" |
| Badge: Not started | — | outline variant, stone-400 text |
| Badge: In progress | — | secondary variant, sky-700 text, sky-100 bg |
| Badge: Complete | — | green-600 text, green-50 bg |
| Badge: Skipped | — | outline variant, stone-400 text, strikethrough domain name |

### Transitions

| Transition | Duration | Easing |
|------------|----------|--------|
| Probe panel open/close (Accordion) | 200ms | ease-out |
| AutoSaveIndicator pulse | 300ms | ease-in-out |
| Domain row hover bg | 150ms | ease |
| Button hover bg | 150ms | ease |
| Question card swap (next/prev) | 150ms fade | opacity 0→1 |
| ResumeBanner dismiss | 200ms slide-up | ease-in |

No animations that could feel jarring in a sensitive conversation context. All transitions are subtle.

### Loading States

| Scenario | Treatment |
|----------|-----------|
| AI probe generation (async, after answer save) | Accordion trigger shows spinner + "Generating suggestions…" text. Does not block navigation. |
| "Begin Interview" (initial question generation) | Button shows spinner + "Preparing interview…". Card content replaced with skeleton: 1 line (question placeholder) + textarea placeholder. |
| Export JSON | Button shows spinner + "Exporting…" for 500ms minimum (prevents flash). |
| Resume from file | File picker → immediate load, no spinner needed (synchronous JSON parse). |

### Error States

| Scenario | Treatment |
|----------|-----------|
| AI probe generation fails | Accordion shows "Suggestions unavailable" in stone-400/14px. No retry button — interviewer can proceed. |
| Initial question generation fails | Error card: "Could not load questions. Check your connection and try again." + "Retry" button. |
| Export fails (localStorage empty) | Toast: "Nothing to export yet. Complete at least one answer first." |
| Resume file invalid JSON | Inline error below file input: "This file doesn't look like a valid session. Please choose a different file." |
| localStorage quota exceeded | Toast: "Storage full. Export your session now to avoid losing data." (destructive variant) |

### Empty States

| Screen | Empty State |
|--------|-------------|
| Interview — no answers yet | Textarea placeholder: "Type the person's answer here…" (stone-400) |
| Probe panel — no probes yet | Not shown until first answer submitted |
| Domain grid — all not started | All 5 domains show "Not started" badge. No special empty state needed. |

## Accessibility

### Keyboard Navigation

| Interaction | Keyboard |
|-------------|----------|
| Tab through setup form fields | Tab / Shift+Tab |
| Submit setup form | Enter on "Begin Interview" button |
| Navigate domain list | Tab to domain row, Enter/Space to select |
| Move between questions | Tab to "Next Question" / "Previous", Enter to activate |
| Open/close probe accordion | Tab to trigger, Enter/Space to toggle |
| Click probe chip | Tab to chip, Enter/Space to copy |
| Export | Tab to "Export session", Enter |
| Dismiss resume banner | Tab to "Resume session" or "Start new session", Enter |

Focus order: Header → ResumeBanner (if present) → Main content → Sidebar (domain list is secondary navigation, not primary flow).

### ARIA

| Element | ARIA |
|---------|------|
| Domain progress sidebar | `<nav aria-label="Interview domains">` |
| Domain row (active) | `aria-current="true"` |
| Domain row (complete) | `aria-label="{Domain Name} — complete"` |
| Domain row (skipped) | `aria-label="{Domain Name} — skipped"` |
| QuestionCard | `<section aria-label="Current question">` |
| Answer textarea | `aria-label="Answer"` + `aria-describedby` pointing to question text id |
| Probe accordion | Radix Accordion handles `aria-expanded`, `aria-controls` natively |
| Probe chip | `role="button"` + `aria-label="Use follow-up: {probe text}"` |
| AutoSaveIndicator | `aria-live="polite"` + `aria-label="Auto-save status"` |
| Loading spinner | `aria-busy="true"` on parent button + `aria-label="Loading"` on spinner |
| Error messages | `role="alert"` for inline errors; `aria-live="assertive"` for storage warning toast |
| Resume banner | `role="banner"` + `aria-label="Previous session detected"` |

### Contrast Ratios

| Pair | Ratio | WCAG Level |
|------|-------|------------|
| stone-900 on #FAFAF9 (body text on bg) | 16.8:1 | AAA |
| stone-900 on #FFFFFF (question text on card) | 19.1:1 | AAA |
| stone-500 on #FAFAF9 (secondary text on bg) | 4.7:1 | AA |
| sky-700 on sky-100 (probe chip text on bg) | 5.2:1 | AA |
| white on sky-500 (button text on accent) | 3.1:1 | AA (large text — button is 14px semibold, qualifies) |
| green-600 on #FAFAF9 (saved indicator) | 4.6:1 | AA |
| red-600 on #FFFFFF (error text on card) | 5.9:1 | AA |

All interactive elements meet minimum 3:1 contrast for non-text (borders, icons).

### Focus Indicators

All focusable elements use: `outline: 2px solid #0EA5E9; outline-offset: 2px`
No `outline: none` without a visible replacement.
