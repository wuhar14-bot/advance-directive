Handlers:
- `handleAnswerChange(value)`: calls `updateAnswer(domain, questionId, questionText, value)` — fires on every keystroke
- `handleNext()`: if answer empty, show error; else advance `currentQuestionIndex`; if last question, set domain status to "complete"; fire probe generation (Plan 04 wires the actual API call — for now, `setProbesLoading(true)` then `setTimeout(() => setProbesLoading(false), 500)` as placeholder)
- `handlePrevious()`: decrement `currentQuestionIndex`
- `handleSelectDomain(domain)`: calls `setCurrentDomain(domain)`; clears probes
- `handleSkipDomain(domain)`: calls `setDomainStatus(domain, "skipped")`; advance to next non-skipped domain

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-01 | Tampering | answer textarea | accept | User types their own content; no server submission in this plan |
| T-03-02 | Information Disclosure | localStorage auto-save | accept | User's own session data; no PII beyond what they entered |
| T-03-03 | Denial of Service | rapid keystroke saves | accept | Zustand set() is synchronous and fast; no debounce needed for localStorage writes at this scale |
