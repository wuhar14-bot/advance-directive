# Plan 05-02 — UI: QuestionCard Multiple-Choice Layout

## Goal
Replace the blank textarea with selectable option cards. "其他" option reveals the textarea for custom input.

## Current State
- `QuestionCard.tsx`: displays question text + Textarea + nav buttons
- `interview/page.tsx`: fetches questions as `{id, text}[]`, passes to QuestionCard

## Target State
- QuestionCard displays 3-4 option cards in a vertical stack
- Tapping an option selects it (sky-500 border, light sky bg), fills the answer
- Last option ("其他") toggles the textarea visible
- If user already typed custom text, "其他" is auto-selected on mount
- Nav buttons work the same way

## Tasks

### Task 1: Update InterviewPage to pass options
File: `src/app/interview/page.tsx`

- Update the `Question` local type (or import from types.ts) to include `options: string[]`
- Pass `options` prop to `QuestionCard`

### Task 2: Redesign QuestionCard
File: `src/components/QuestionCard.tsx`

New layout:
```
┌─────────────────────────────┐
│  问题文本 (24px semibold)    │
│                             │
│  ┌─────────────────────┐    │
│  │ ○ 选项A文本          │    │  ← tappable card, rounded-xl
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ ○ 选项B文本          │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ ○ 选项C文本          │    │
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │
│  │ ✎ 其他，我想说…      │    │  ← "其他" option, different icon
│  └─────────────────────┘    │
│                             │
│  ┌─────────────────────┐    │  ← only visible when "其他" selected
│  │ Textarea for custom  │    │
│  │ answer               │    │
│  └─────────────────────┘    │
│                             │
│  [上一题]  [下一题]          │
└─────────────────────────────┘
```

State management:
- `selectedIndex: number | null` — which option is selected (-1 for "其他")
- `isCustom: boolean` — derived from selectedIndex === -1 or last option selected
- `customText: string` — the textarea value when in custom mode

Behavior:
- Click option → `setSelectedIndex(i)`, call `onAnswerChange(options[i])`
- Click "其他" → `setSelectedIndex(-1)`, `setIsCustom(true)`, focus textarea
- Type in textarea → `onAnswerChange(customText)`
- On mount: if `answer` matches an option, pre-select it; if answer exists but doesn't match any option, show "其他" + textarea with the existing text

### Task 3: Styling
- Unselected option: `border border-stone-200 bg-white rounded-xl p-4 cursor-pointer hover:border-stone-300`
- Selected option: `border-2 border-sky-500 bg-sky-50 rounded-xl p-4`
- "其他" option: same styling but with a pencil icon (✎) instead of circle
- Transition: `transition-all duration-150`
- The textarea appears with a slide-down animation when "其他" is selected

### Task 4: Accessibility
- Options use `role="radiogroup"` with `role="radio"` per option
- `aria-checked` on selected option
- Keyboard: arrow keys to navigate options, Enter/Space to select
- Textarea gets `aria-label="自定义回答"`

## Verification
- Selecting an option fills the answer and enables "下一题"
- "其他" reveals textarea, typing fills the answer
- Going back to a question restores the previously selected option
- Custom text answers are preserved when navigating back
- Archive generation works identically with option-selected answers
