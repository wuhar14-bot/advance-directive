# Plan 05-01 — API: Extend Question Generation with Answer Options

## Goal
Update the `/api/interview/questions` route to return 3-4 answer options alongside each question.

## Current State
- Route: `src/app/api/interview/questions/route.ts`
- Returns: `{ questions: [{id, text}] }`
- Prompt asks for 4 open-ended questions in JSON array format

## Target State
- Returns: `{ questions: [{id, text, options}] }` where `options` is `string[]` of 3-4 choices
- Last option is always a variant of "其他情况" or "以上都不太符合" to signal the escape hatch
- Prompt updated to generate contextually appropriate options per question

## Tasks

### Task 1: Update types
File: `src/lib/types.ts`

Add a `Question` interface (currently questions are untyped inline objects):
```ts
interface Question {
  id: string
  text: string
  options: string[]
}
```

### Task 2: Update API prompt
File: `src/app/api/interview/questions/route.ts`

Update the system prompt and user prompt to instruct the AI to generate options:
- Each question gets 3 specific options + 1 "其他" option
- Options should be concise (under 30 chars each), written as first-person statements from the elderly person's perspective
- Example: question about medical preferences → options like "我希望保守治疗", "我愿意尝试手术", "我想先听医生建议再决定"
- The "其他" option text should be "以上都不太符合，我想说…"

Update the JSON format in the prompt:
```json
[
  {
    "id": "q1",
    "text": "关于日常用药，您有什么想法？",
    "options": ["我一直按时吃药，很重视", "我觉得少吃药比较好", "我听医生的安排", "以上都不太符合，我想说…"]
  }
]
```

### Task 3: Validate response parsing
The existing regex `text.match(/\[[\s\S]*\]/)` should still work since the structure is still a JSON array. Add a fallback: if parsed questions lack `options`, inject a default `["其他"]` so the UI always has at least the escape hatch.

## Verification
- API returns questions with 3-4 options each
- Options are contextually relevant to the domain and question
- Fallback works if AI omits options
- No breaking changes to existing transcript storage
